# GIF Support and Keyboard Navigation Design

**Date:** 2026-04-04
**Status:** Approved
**Author:** Claude (with user collaboration)

## Overview

Add a third "Gifs" section to the artwork gallery, displaying animated GIF thumbnails from the `gif/` directory. Add navigation buttons to the top bar for quick access to all three sections. Implement full keyboard navigation for accessibility.

## Requirements

### Functional Requirements

1. **Three-section gallery:**
   - Artwork (showcase/)
   - Other Artwork (png/)
   - Gifs (gif/)

2. **Section order:** Artwork → Other Artwork → Gifs

3. **GIF behavior:**
   - Animate automatically in thumbnails
   - Loop continuously in lightbox
   - Show same metadata as images (dimensions × file size)

4. **Navigation:**
   - [Back] button → larger, logo-sized
   - Section navigation buttons → same size as current [Back] button
   - Jump to each section via anchor links

5. **Empty sections:**
   - Show "No artwork yet" message
   - Keep navigation button visible

6. **Keyboard navigation:**
   - All interactive elements must be focusable
   - Visible focus indicators
   - Logical tab order
   - Keyboard shortcuts for lightbox navigation
   - Skip link to main content

### 4. JavaScript (`website/page.js`)

#### Keyboard Navigation

```javascript
// Keyboard shortcuts for lightbox
document.addEventListener('keydown', (e) => {
    const lightbox = document.getElementById('lightbox');
    if (!lightbox.classList.contains('active')) return;

    switch(e.key) {
        case 'Escape':
            closeLightbox();
            break;
        case 'ArrowLeft':
            showPrevImage();
            break;
        case 'ArrowRight':
            showNextImage();
            break;
        case 'Home':
            showImage(0);
            e.preventDefault();
            break;
        case 'End':
            showImage(imageData.length - 1);
            e.preventDefault();
            break;
    }
});

// Focus trap in lightbox
function trapFocus() {
    const lightbox = document.getElementById('lightbox');
    const focusableElements = lightbox.querySelectorAll(
        'button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])'
    );
    const firstElement = focusableElements[0];
    const lastElement = focusableElements[focusableElements.length - 1];

    lightbox.addEventListener('keydown', (e) => {
        if (e.key !== 'Tab') return;

        if (e.shiftKey && document.activeElement === firstElement) {
            e.preventDefault();
            lastElement.focus();
        } else if (!e.shiftKey && document.activeElement === lastElement) {
            e.preventDefault();
            firstElement.focus();
        }
    });
}
```

#### Restore Focus on Close

```javascript
let lastFocusedElement = null;

function openLightbox(index) {
    lastFocusedElement = document.activeElement;
    // ... existing code
    trapFocus();
}

function closeLightbox() {
    // ... existing code
    if (lastFocusedElement) {
        lastFocusedElement.focus();
    }
}
```

## Data Flow

1. `collect_images()` walks showcase/, png/, gif/ directories
2. Images sorted into three buckets by source directory
3. `generate_gallery_html()` creates HTML for each section with proper IDs
4. `generate_navigation_buttons()` creates nav links
5. Lightbox offsets calculated: showcase (0-N), other (N-M), gifs (M-P)
6. Template populated with gallery HTML + navigation buttons
7. `index.html` written with full keyboard navigation support

## Error Handling

- **Missing directories:** Silently skip (already implemented)
- **Unreadable images:** Warning message, skip that file (already implemented)
- **Empty sections:** Show "No artwork yet" message, keep nav button
- **Invalid keyboard input:** Ignore unhandled keys in lightbox

## Browser Behavior

- GIF thumbnails animate automatically (browser default)
- Lightbox opens GIFs with continuous looping
- Navigation buttons smooth-scroll to sections
- All metadata displayed consistently
- Focus management follows WAI-ARIA best practices

## Accessibility

- **WCAG 2.1 Level AA** compliance
- **Keyboard navigation:** All functionality available via keyboard
- **Focus indicators:** Visible on all interactive elements
- **ARIA attributes:** Proper roles and labels on dynamic elements
- **Skip link:** Jump to main content
- **Screen reader:** Semantic HTML + ARIA labels
- **Motion:** Respect `prefers-reduced-motion` preference
